Computes the shortest path between two nodes using Dijkstra's algorithm.

Finds the shortest path from the source to the destination while avoiding specified nodes and segments. The algorithm runs in O((V + E) log V), where V is the number of vertices and E is the number of edges.

Parameters
graphThe graph to traverse.
sourceThe ID of the source node.
destThe ID of the destination node.
avoidNodesA list of node IDs to avoid.
avoidSegmentsA list of edge pairs (source, destination) to avoid.
isDrivingIndicates whether the path is for driving. Defaults to true.
Returns
A vector of LocationInfo representing the shortest path.

Computes the driving route between two nodes.

Finds the shortest driving route from the source to the destination, optionally passing through an intermediate node. The algorithm runs in O((V + E) log V) for each segment of the route (source to intermediate and intermediate to destination), where:

  • V is the number of vertices.
  • E is the number of edges.
Parameters
graphThe graph to traverse.
sourceThe ID of the source node.
destThe ID of the destination node.
avoidNodesA list of node IDs to avoid.
avoidSegmentsA list of edge pairs (source, destination) to avoid.
includeNodeAn optional intermediate node to include in the route. Defaults to -1.
Returns
A vector of LocationInfo representing the driving route.

Finds all parking nodes reachable from the source.

Identifies parking nodes and computes the driving path to each of them. The algorithm runs in O((V + E) log V), where V is the number of vertices and E is the number of edges.

sourceThe ID of the source node.
avoidNodesA list of node IDs to avoid.

A vector of pairs, each containing a parking node and the path to it.

Retrieves the path from the origin to the destination.

Constructs the path by backtracking from the destination to the origin using the shortest path tree. This function runs in O(V), where V is the number of vertices in the path.
